# 《无名之界》

一个由 LLM 驱动的命令行神明模拟器。你扮演一个正在注视世界的存在，从石器时代开始，见证文明的生长与消亡。

---

## 世界观

这个世界建立在两种根本底质之上：**凝质（Sang）**，构成一切有形之物；**息（Veth）**，渗透万物的意识场域。两者并非对立，而是同一实在的两面。科技与魔法在这里不是分类，而是同一力量的不同读取方式。

世界从石器时代起步。没有可参考的历史，没有已知的神话。一切都在发生之中。

---

## 设计理念：极简内核 + 自生长法则

**硬编码的部分只有世界的最小物理：** 时间、人、信仰循环、玩家干预通道、NPC 自主行为、涌现引擎。

**其他一切都由 LLM 在运行中自行长出来。** 变异、神话、异人、其他神明、贸易、战争、瘟疫、仪式……这些不是预先内置的 feature，而是涌现引擎根据世界积累的事件判断条件后，自动生成对应模块并热加载的结果。

每次运行的世界都不一样——你见到的系统是这个世界因为你而长出来的。

---

## 特性

- **每个命名居民都有完整的个人时间线** — 出生、相遇、祈祷、自发行为、死亡，全部记录
- **真实流逝的世界时间** — 后台线程推进年份；离线期间时间也在流逝，回来时自动补偿
- **信仰双向闭环** — 神力是你的能量；居民的信仰加速积累；你的回应改变他们对你的认知
- **NPC 自主权** — 居民有自己的性格、记忆碎片，会自发做事，甚至向神明提出改变世界的提案
- **神明自询** — 以你所见为据回溯世界脉络，未曾见过的无法回答。与凡人的直接沟通不是内置能力——需要世界自己长出能听见你的耳朵
- **世界故事切片** — 随时查看当下正在发生的事
- **自生长法则** — 涌现引擎检测到新兴系统时，LLM 自动生成 Python 模块并热加载；崩溃自动修复；居民可提出升级

## 安装

```bash
pip install openai python-dotenv rich
```

复制 `.env.example` 为 `.env`，填入你的 [OpenRouter](https://openrouter.ai) API Key：

```
OPENROUTER_API_KEY=sk-or-v1-你的key
```

---

## 启动

```bash
python main.py
```

首次运行自动创世。之后每次启动读取存档，世界继续。

---

## 指令

支持中英文双输入。

| 指令 | 英文 | 说明 |
|------|------|------|
| `赐予 <物品>` | `gift` | 消耗 10 神力，向部落赐予一件物品（赐予后部落需时间消化） |
| `施放 <奇迹>` | `cast` | 消耗 50 神力，降下神迹 |
| `回应 <名字> <类型>` | `respond` | 回应祈祷：答应 / 无视 / 惩戒 / 赐福 |
| `凝视 <名字或地点>` | `gaze` | 深度洞察某人或某地 |
| `问 <问题>` | `ask` | 神明自询，以所见为据回溯世界脉络 |
| `故事` | `story` | 查看当下正在发生的世界切片 |
| `状态` | `status` / `s` | 世界概况（神力、人口、法则） |
| `地图` | `map` / `m` | 世界感知图 |
| `人群` | `people` | 命名居民列表 |
| `人 <名字>` | `person` | 某人的完整人生时间线 |
| `祈祷` | `prayers` | 等待回应的祈祷列表 |
| `法则` | `weave` | 世界自然涌现的法则 |
| `帮助` | `help` | 显示指令列表 |
| `退出` | `quit` | 离开（世界继续流逝） |

## 架构

```
core/            世界最小物理：WorldState、Person、PopulationPool、Simulator
systems/         硬编码 tick：人口（生老病死、祈祷）+ NPC 自主
llm/             LLM 通道：bible、schemas、action、divine、codegen、autonomy
evolution/       涌现引擎：ModuleLoader（热加载）+ MetaSystem（涌现检测）
modules/         运行时由 LLM 生成的法则模块（首次运行为空）
cli/             rich 终端渲染
state_manager.py 薄协调器
main.py          入口与指令循环
```

涌现引擎每 3 年检查一次：读近期事件 + 世界状态，对照「涌现类型调色板」（世界物理 / 生命与个体 / 社会结构 / 精神与文化 / 超自然），判断是否该有新系统出现。若是，调 Sonnet 生成对应模块代码，写入 `modules/` 并热加载。

模块是**纯叠加性**的——可以读所有状态、添加事件、在 `module_data` 里持久化数据，但不会覆盖核心 tick 逻辑。

## 技术栈

- Python 3.11+
- OpenRouter API（claude-haiku-4-5 日常调用，claude-sonnet-4-5 代码生成）
- `rich` 终端可视化
- `threading` 后台世界模拟
- `exec()` 热加载 LLM 生成的法则模块
